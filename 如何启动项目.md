# 开放式实验室网上预约管理系统 — 完整启动与开发指南

## 项目概览

这是一个 **「开放式实验室网上预约管理系统」**，基于若依（RuoYi）前后端分离框架 v3.8.9 二次开发：

| 层级 | 技术栈 | 版本 | 位置 |
|------|--------|------|------|
| 后端 | Spring Boot + MyBatis + Maven | 2.5.15 | `back/labs/` |
| 前端 | Vue 3 + Element Plus + Vite | 3.4.0 / 2.4.3 / 5.0.4 | `front/RuoYi-Vue3/` |
| 数据库 | MySQL | 8.0.27 (兼容5.7) | localhost:3306 |
| 缓存 | Redis | 6.x+ (Windows版可用) | localhost:6379 |
| 安全 | Spring Security + JWT | 5.7.12 / jjwt 0.9.1 | — |
| 连接池 | Druid (阿里) | 1.2.23 | — |
| JDK | Amazon Corretto / OpenJDK | 21 (编译目标1.8) | — |

## 二、配置文件说明

### 2.1 后端核心配置

#### 数据库连接 (`back/labs/labs-admin/src/main/resources/application-druid.yml`)

```yaml
spring:
  datasource:
    druid:
      master:
        url: jdbc:mysql://localhost:3306/open_lab_reservation?useUnicode=true&characterEncoding=utf8&zeroDateTimeBehavior=convertToNull&useSSL=true&serverTimezone=GMT%2B8
        username: root        # ← 修改为你的MySQL用户名
        password: root        # ← 修改为你的MySQL密码
```

#### 应用配置 (`back/labs/labs-admin/src/main/resources/application.yml`)

| 配置项 | 默认值 | 说明 | 修改场景 |
|--------|--------|------|----------|
| `server.port` | 8080 | 后端端口 | 端口冲突时修改 |
| `spring.redis.host` | localhost | Redis地址 | Redis在其他机器时 |
| `spring.redis.port` | 6379 | Redis端口 | 非默认端口时 |
| `spring.redis.password` | (空) | Redis密码 | 生产环境必设 |
| `token.expireTime` | 30 | Token有效期(分钟) | 测试时可延长 |
| `ruoyi.captchaType` | math | 验证码类型 | math/char |
| `ruoyi.profile` | H:/uploadPath | 文件上传路径 | 改为实际路径 |

### 2.2 前端配置

#### 环境变量 (`front/RuoYi-Vue3/.env.development`)

```env
# 开发环境配置
VITE_APP_BASE_API = '/dev-api'
VITE_APP_TITLE = '开放式实验室预约系统'
```

#### Vite配置 (`front/RuoYi-Vue3/vite.config.js`)

```javascript
server: {
  host: '0.0.0.0',
  port: 81,        // ← 前端端口，冲突时修改
  proxy: {
    '/dev-api': {
      target: 'http://localhost:8080',  // ← 后端地址
      changeOrigin: true,
      rewrite: (p) => p.replace(/^\/dev-api/, '')
    }
  }
}
```


## 五、常见问题与解决方案

### 5.1 后端启动问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| `Connection refused: localhost:3306` | MySQL未启动 | `net start MySQL80` |
| `Access denied for user 'root'` | 密码错误 | 修改 `application-druid.yml` 中的password |
| `Unknown database 'open_lab_reservation'` | 未创建数据库 | 执行CREATE DATABASE语句 |
| `RedisConnectionFailureException` | Redis未启动 | 启动Redis服务 |
| `Port 8080 already in use` | 端口被占用 | `netstat -ano \| findstr 8080` 查找并关闭占用进程 |
| `java: 无效的源发行版: 21` | IDE中JDK版本不匹配 | 检查Project Structure中的SDK设置 |
| `Cannot find symbol: javax.servlet…` | 缺少Servlet API | IDEA: File→Project Structure→Modules→添加Web |

### 5.2 前端启动问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| `npm install` 失败 | 网络问题/Node版本 | 设置国内镜像: `npm config set registry https://registry.npmmirror.com` |
| `Error: Cannot find module 'vite'` | 依赖未安装 | 重新执行 `npm install` |
| `Port 81 already in use` | 端口冲突 | 修改 `vite.config.js` 中 `server.port` |
| 页面空白/404 | 前端代理未生效 | 检查vite.config.js中proxy配置 |
| `Mixed Content` 错误 | HTTP/HTTPS混用 | 确保前后端都使用HTTP |

### 5.3 运行时问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 登录后立即跳回登录页 | Token失效 | 检查Redis是否正常运行 |
| 验证码不显示 | 字体缺失 | 检查Java是否支持AWT（无头模式问题） |
| 操作提示"权限不足" | 角色权限未配置 | 检查sys_role_menu表 |
| 文件上传失败 | 上传路径不存在 | 创建 `H:/uploadPath` 目录或修改配置 |
| 数据库中文乱码 | 字符集问题 | 数据库使用utf8mb4，连接串指定characterEncoding=utf8 |

### 5.4 快速诊断脚本

```bash
# 环境诊断
echo "=== JDK ===" && java -version
echo "=== Maven ===" && mvn -version 2>&1 | head -3
echo "=== Node ===" && node -v && npm -v
echo "=== MySQL ===" && mysql -u root -proot -e "SELECT 1" 2>&1
echo "=== Redis ===" && redis-cli ping 2>&1
echo "=== 后端端口 ===" && netstat -ano | grep 8080
echo "=== 前端端口 ===" && netstat -ano | grep ":81 "
```

## 八、部署指南

### 8.1 后端部署（生产环境）

```bash
# 1. 打包
cd back/labs
mvn clean package -DskipTests

# 2. 上传jar包到服务器
scp labs-admin/target/labs-admin.jar user@server:/opt/labs/

# 3. 启动（使用外置配置文件）
java -jar /opt/labs/labs-admin.jar \
  --spring.config.location=/opt/labs/application-prod.yml \
  --spring.profiles.active=prod

# 4. 系统服务（Linux systemd）
sudo vim /etc/systemd/system/labs.service
```

```ini
[Unit]
Description=Open Lab Reservation System
After=network.target mysql.service redis.service

[Service]
User=labs
WorkingDirectory=/opt/labs
ExecStart=/usr/bin/java -jar /opt/labs/labs-admin.jar --spring.profiles.active=prod
Restart=on-failure
RestartSec=10

[Install]
WantedBy=multi-user.target
```

### 8.2 前端部署

```bash
# 1. 构建生产版本
cd front/RuoYi-Vue3
npm run build:prod

# 2. 部署dist目录到Nginx
scp -r dist/* user@server:/var/www/labs/

# 3. Nginx配置
sudo vim /etc/nginx/sites-available/labs
```

```nginx
server {
    listen 80;
    server_name labs.example.com;
    root /var/www/labs;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    location /prod-api/ {
        proxy_pass http://127.0.0.1:8080/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```
