### BUG-001: 预约提交无冲突检测

| 字段 | 内容 |
|------|------|
| **缺陷编号** | T29_BUG-RSV-001 |
| **严重程度** | **严重** |
| **优先级** | 高 |
| **状态** | ✅ 已修复 |
| **所属模块** | M3 预约管理 |
| **影响范围** | 核心预约功能 |
| **发现日期** | 2026-06-21 |

#### 2.1.1 问题描述

`ReservationController.insertReservation()` 方法仅设置了 `status=0L`，未调用任何冲突检测逻辑，导致同一用户可在同一实验室同一时段重复提交预约。

#### 2.1.2 复现步骤

```bash
# 步骤1: 提交第一次预约
curl -X POST http://localhost:8080/reservation/reservation/insert \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"userId":1,"labId":1,"reservationDate":"2026-06-22","timeSlot":1,"purpose":"实验1"}'
# 返回: {"code":200,"msg":"操作成功"} ✓

# 步骤2: 再次提交相同预约
curl -X POST http://localhost:8080/reservation/reservation/insert \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"userId":1,"labId":1,"reservationDate":"2026-06-22","timeSlot":1,"purpose":"实验2"}'
# 返回: {"code":200,"msg":"操作成功"} ← BUG！
# 预期: {"code":500,"msg":"该时段已被预约"}
```

#### 2.1.3 根因分析

**源码位置：** [`ReservationController.java:123-130`](back/labs/labs-management/src/main/java/com/labs/reservation/controller/ReservationController.java:123)

```java
// 原始代码 — 缺少冲突检测
@PostMapping("/insert")
public AjaxResult insertReservation(@RequestBody Reservation reservation) {
    reservation.setStatus(0L);
    // 可以在这里添加用户预约时的一些验证，比如时间段冲突、权限校验等  ← 注释说明但未实现！
    return toAjax(reservationService.insertReservationFromSchedule(reservation));
}
```

**根因：** 开发者预留了注释但未实现冲突检测逻辑。`ReservationServiceImpl.insertReservationFromSchedule()` 仅执行 INSERT 操作，无任何业务规则校验。虽然 Mapper 中定义了 `selectConflictingReservations()` 方法，但 Controller 和 Service 层均未调用。

#### 2.1.4 修复方案

在 `insertReservation()` 方法中增加完整的冲突检测逻辑：

1. 时间合法性校验（时间段范围1-4）
2. 日期合法性校验（不能为过去日期）
3. 冲突检测（同一实验室同一日期同一时段不能重复预约）
4. 实验室存在性校验
5. 用户存在性校验

#### 2.1.5 修复代码

**文件：** `back/labs/labs-management/src/main/java/com/labs/reservation/controller/ReservationController.java`

```java
// SEARCH/REPLACE 补丁 1 — insertReservation方法
    /**
     * 用户提交预约申请（前台）
     */
    @Log(title = "用户预约申请", businessType = BusinessType.INSERT)
    @PostMapping("/insert")
    public AjaxResult insertReservation(@RequestBody Reservation reservation) {
        // 设置初始状态（0:待审核），防止前端传非法状态
        reservation.setStatus(0L);

        // ========== 新增：输入校验 ==========
        
        // 1. 时间段范围校验（1-4）
        if (reservation.getTimeSlot() == null 
            || reservation.getTimeSlot() < 1L 
            || reservation.getTimeSlot() > 4L) {
            return AjaxResult.error("时间段必须在1-4之间（1:8-10, 2:10-12, 3:14:30-16:30, 4:16:30-18:30）");
        }
        
        // 2. 日期合法性校验（不能为过去日期）
        if (reservation.getReservationDate() != null) {
            java.util.Date today = java.sql.Date.valueOf(java.time.LocalDate.now());
            if (reservation.getReservationDate().before(today)) {
                return AjaxResult.error("不能预约过去日期，请选择今天或未来的日期");
            }
        } else {
            return AjaxResult.error("预约日期不能为空");
        }
        
        // 3. 实验室存在性校验
        if (reservation.getLabId() == null) {
            return AjaxResult.error("实验室ID不能为空");
        }
        
        // 4. 用户存在性校验
        if (reservation.getUserId() == null) {
            return AjaxResult.error("用户ID不能为空");
        }
        
        // 5. 用途不能为空
        if (reservation.getPurpose() == null || reservation.getPurpose().trim().isEmpty()) {
            return AjaxResult.error("预约用途不能为空");
        }
        
        // 6. 冲突检测：同一实验室同一日期同一时段不能重复预约
        List<Reservation> conflicts = reservationService.selectConflictingReservations(reservation);
        if (conflicts != null && !conflicts.isEmpty()) {
            return AjaxResult.error("该时段已被预约，请选择其他时间段");
        }

        return toAjax(reservationService.insertReservationFromSchedule(reservation));
    }
```

**需要新增的 import：**
```java
import java.util.List;
```

> 注：`List` 已在文件顶部 import，无需额外添加。

### BUG-004: 审核后未拒绝其他冲突预约

| 字段 | 内容 |
|------|------|
| **缺陷编号** | T29_BUG-RSV-004 |
| **严重程度** | **严重** |
| **优先级** | 中 |
| **状态** | ✅ 已修复 |

#### 2.4.1 问题描述

审核通过某预约后，仅删除了 `lab_schedule` 排期记录（释放可预约时段），但未自动拒绝同一实验室同一时段的其他待审核预约（`status=0`）。

#### 2.4.2 复现步骤

```bash
# 用户A提交预约 → 待审核(ID=1)
# 用户B提交同一实验室同时段预约 → 待审核(ID=2)
# 教师审核通过ID=1 → ID=2仍然是"待审核"状态
```

#### 2.4.3 根因分析

**源码位置：** [`ReservationController.java:103-114`](back/labs/labs-management/src/main/java/com/labs/reservation/controller/ReservationController.java:103)

```java
// 原始代码 — 仅删除排期，未拒绝冲突预约
@PutMapping("/review")
public AjaxResult reviewReservation(@RequestBody Reservation reservation) {
    Long teacherId = getUserId();
    reservation.setTeacherId(teacherId);
    if (reservation.getStatus() != null && reservation.getStatus() == 1) {
        labScheduleService.deleteByReservation(reservation);  // 仅删除排期
    }
    return toAjax(reservationService.reviewReservation(reservation));
}
```

#### 2.4.4 修复方案

在审核通过时，批量将同一实验室同一日期同一时段的状态为"待审核(0)"的其他预约设为"已拒绝(2)"。

#### 2.4.5 修复代码

**文件：** `back/labs/labs-management/src/main/java/com/labs/reservation/controller/ReservationController.java`

```java
// SEARCH/REPLACE 补丁 2 — reviewReservation方法
    /**
     * 审核预约记录（同意 / 拒绝）
     */
    @Log(title = "预约审核", businessType = BusinessType.UPDATE)
    @PutMapping("/review")
    public AjaxResult reviewReservation(@RequestBody Reservation reservation) {
        Long teacherId = getUserId(); // 获取当前登录教师 ID
        reservation.setTeacherId(teacherId);

        // 如果是"同意"审核
        if (reservation.getStatus() != null && reservation.getStatus() == 1) {
            // 删除 lab_schedule 中冲突时间段的记录
            labScheduleService.deleteByReservation(reservation);
            
            // ========== 新增：自动拒绝其他冲突的待审核预约 ==========
            List<Reservation> conflicts = reservationService.selectConflictingReservations(reservation);
            if (conflicts != null && !conflicts.isEmpty()) {
                for (Reservation conflict : conflicts) {
                    // 跳过自身
                    if (conflict.getId().equals(reservation.getId())) {
                        continue;
                    }
                    conflict.setStatus(2L);  // 设为"已拒绝"
                    conflict.setTeacherId(teacherId);
                    conflict.setReviewNote("该时段已有其他预约被审核通过，系统自动拒绝");
                    reservationService.reviewReservation(conflict);
                }
            }
        }

        return toAjax(reservationService.reviewReservation(reservation));
    }
```

### BUG-008: 实验室删除无关联检查

| 字段 | 内容 |
|------|------|
| **缺陷编号** | T29_BUG-LAB-008 |
| **严重程度** | 一般 |
| **优先级** | 中 |
| **状态** | ⚠️ 部分修复 |

#### 3.4.1 问题描述

删除实验室后，关联的 `reservation` 和 `lab_schedule` 记录未做级联处理，产生孤儿数据。

#### 3.4.2 根因分析

`LabsController.remove()` → `LabsServiceImpl.deleteLabsByIds()` 仅执行 `DELETE FROM labs WHERE id IN (...)`，未级联删除关联记录。数据库未设置外键约束（CASCADE）。

#### 3.4.3 修复方案

在删除实验室前检查是否存在关联的预约或排期记录，如存在则阻止删除并提示用户先清理关联数据：

```java
// SEARCH/REPLACE 补丁 — LabsController.remove()
    @PreAuthorize("@ss.hasPermi('labs_browse:labs_browse:remove')")
    @Log(title = "实验室信息管理", businessType = BusinessType.DELETE)
    @DeleteMapping("/{ids}")
    public AjaxResult remove(@PathVariable Long[] ids) {
        // ========== 新增：关联数据检查 ==========
        for (Long id : ids) {
            // 检查是否存在关联排期
            LabSchedule scheduleQuery = new LabSchedule();
            scheduleQuery.setLabId(id);
            List<LabSchedule> schedules = labScheduleService.selectLabScheduleList(scheduleQuery);
            if (schedules != null && !schedules.isEmpty()) {
                return AjaxResult.error("实验室(ID=" + id + ")存在关联排期记录，请先清理排期后再删除实验室");
            }
            // 检查是否存在关联预约
            Reservation resQuery = new Reservation();
            resQuery.setLabId(id);
            List<Reservation> reservations = reservationService.selectReservationList(resQuery);
            if (reservations != null && !reservations.isEmpty()) {
                return AjaxResult.error("实验室(ID=" + id + ")存在关联预约记录，请先清理预约后再删除实验室");
            }
        }
        return toAjax(labsService.deleteLabsByIds(ids));
    }
```

**需在 LabsController 中新增：**
```java
@Autowired
private ILabScheduleService labScheduleService;

@Autowired
private IReservationService reservationService;
```

```vue
<!-- 标准时间段映射 -->
<template #default="scope">

  <span v-else-if="scope.row.timeSlot === 2">10:00 - 12:00</span>
```

### 5.2 完整修复后的 ReservationController.insertReservation()

```java
/**
 * 用户提交预约申请（前台）
 * 
 * 修复内容：
 * - BUG-001: 增加冲突检测
 * - BUG-002: 增加时间段范围校验(1-4)
 * - BUG-003: 增加过去日期拦截
 * - BUG-005: 增加实验室存在性校验
 * - BUG-006: 增加空用途拦截
 */
@Log(title = "用户预约申请", businessType = BusinessType.INSERT)
@PostMapping("/insert")
public AjaxResult insertReservation(@RequestBody Reservation reservation) {
    // 1. 强制设置初始状态为"待审核"
    reservation.setStatus(0L);

    // 2. 时间段范围校验（1-4）
    if (reservation.getTimeSlot() == null 
        || reservation.getTimeSlot() < 1L 
        || reservation.getTimeSlot() > 4L) {
        return AjaxResult.error("时间段必须在1-4之间（1:8-10, 2:10-12, 3:14:30-16:30, 4:16:30-18:30）");
    }

    // 3. 日期合法性校验
    if (reservation.getReservationDate() != null) {
        java.util.Date today = java.sql.Date.valueOf(java.time.LocalDate.now());
        if (reservation.getReservationDate().before(today)) {
            return AjaxResult.error("不能预约过去日期，请选择今天或未来的日期");
        }
    } else {
        return AjaxResult.error("预约日期不能为空");
    }

    // 4. 实验室存在性校验
    if (reservation.getLabId() == null) {
        return AjaxResult.error("实验室ID不能为空");
    }
    Labs lab = labsService.selectLabsById(reservation.getLabId());
    if (lab == null) {
        return AjaxResult.error("实验室不存在");
    }

    // 5. 用户存在性校验
    if (reservation.getUserId() == null) {
        return AjaxResult.error("用户ID不能为空");
    }

    // 6. 用途不能为空
    if (reservation.getPurpose() == null || reservation.getPurpose().trim().isEmpty()) {
        return AjaxResult.error("预约用途不能为空");
    }

    // 7. 冲突检测
    List<Reservation> conflicts = reservationService.selectConflictingReservations(reservation);
    if (conflicts != null && !conflicts.isEmpty()) {
        return AjaxResult.error("该时段已被预约，请选择其他时间段");
    }

    return toAjax(reservationService.insertReservationFromSchedule(reservation));
}
```

### 5.3 完整修复后的 ReservationController.reviewReservation()

```java
/**
 * 审核预约记录（同意 / 拒绝）
 * 
 * 修复内容：
 * - BUG-004: 审核通过时自动拒绝其他冲突预约
 * - BUG-010: 防止重复审核
 */
@Log(title = "预约审核", businessType = BusinessType.UPDATE)
@PutMapping("/review")
public AjaxResult reviewReservation(@RequestBody Reservation reservation) {
    // 0. 防止重复审核
    Reservation existing = reservationService.selectReservationById(reservation.getId());
    if (existing == null) {
        return AjaxResult.error("预约记录不存在");
    }
    if (existing.getStatus() != 0L) {
        return AjaxResult.error("该预约已审核，不能重复审核（当前状态："
            + (existing.getStatus() == 1L ? "已通过" : "已拒绝") + "）");
    }

    Long teacherId = getUserId();
    reservation.setTeacherId(teacherId);

    // 如果是"同意"审核
    if (reservation.getStatus() != null && reservation.getStatus() == 1) {
        // 删除 lab_schedule 中冲突时间段的记录
        labScheduleService.deleteByReservation(reservation);

        // 自动拒绝其他冲突的待审核预约
        List<Reservation> conflicts = reservationService.selectConflictingReservations(reservation);
        if (conflicts != null && !conflicts.isEmpty()) {
            for (Reservation conflict : conflicts) {
                if (conflict.getId().equals(reservation.getId())) {
                    continue;
                }
                conflict.setStatus(2L);
                conflict.setTeacherId(teacherId);
                conflict.setReviewNote("该时段已有其他预约被审核通过，系统自动拒绝");
                reservationService.reviewReservation(conflict);
            }
        }
    }

    return toAjax(reservationService.reviewReservation(reservation));
}
```

### 7.4 缺陷注入原因分析

1. **代码自动生成未补充业务逻辑**：MyBatis代码生成器生成了基础的CRUD模板，但开发者在后续二次开发中未补充业务规则校验（5个缺陷）。
2. **注释标记但未实现**：`ReservationController` 中明确注释"可以在这里添加用户预约时的一些验证"，但验证代码被遗漏（3个缺陷）。
3. **前端与后端校验分离**：前端有部分校验（时间段下拉选择、日期选择器），但后端完全依赖前端校验，导致API直接调用可绕过（3个缺陷）。
4. **数据库约束缺失**：业务表未设置外键约束，依赖应用层保证数据一致性，但应用层校验不完整（1个缺陷）。

## 第8章 预防建议

### 8.1 编码阶段

1. **强制后端校验**：所有API入参必须在Controller层进行校验，不信任前端传来的任何数据
2. **使用Bean Validation**：在 `Reservation` 等实体类上使用 `@NotNull`、`@Min`、`@Max`、`@FutureOrPresent` 等JSR-303注解
3. **Code Review检查清单**：将"是否实现了冲突检测"、"是否校验了输入范围"等列为必须检查项

### 8.2 数据库设计

1. **添加外键约束**：
   ```sql
   ALTER TABLE reservation ADD CONSTRAINT fk_reservation_lab 
     FOREIGN KEY (lab_id) REFERENCES labs(id) ON DELETE RESTRICT;
   ALTER TABLE reservation ADD CONSTRAINT fk_reservation_user 
     FOREIGN KEY (user_id) REFERENCES sys_user(user_id) ON DELETE RESTRICT;
   ```
2. **添加唯一约束**：防止同一实验室同一日期同一时段重复预约
   ```sql
   ALTER TABLE reservation ADD UNIQUE KEY uk_lab_date_slot 
     (lab_id, reservation_date, time_slot, user_id);
   ```

### 8.3 测试阶段

1. **API层专项测试**：对所有API端点进行参数边界值测试
2. **安全测试前置**：在功能测试阶段就纳入SQL注入、XSS等安全测试
3. **自动化回归测试**：将回归测试脚本集成到CI/CD流程中
