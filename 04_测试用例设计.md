## 1. 测试用例设计概述

### 1.1 设计目标

为开放式实验室网上预约管理系统的每一个测试需求确定测试用例集（Test Case Suite），确保测试覆盖以下维度：

| 维度 | 覆盖目标 | 用例数 |
|------|----------|--------|
| 功能正确性 | 所有模块CRUD功能正常 | 15 |
| 输入校验 | 必填项/边界值/特殊字符/SQL注入 | 12 |
| 业务规则 | 预约冲突/时间合法性/状态流转 | 8 |
| 用户界面 | 布局/控件/提示信息 | 5 |
| 数据库完整性 | CRUD/事务/一致性 | 8 |
| API接口 | 正常/异常/认证鉴权 | 8 |
| 白盒覆盖 | 语句/判定/条件/路径 | 4 |
| 单元测试 | 方法级正确性 | 10 |
| 性能 | 并发/响应时间/吞吐量 | 4 |
| 安全性 | 认证/注入/越权 | 3 |
| 兼容性 | 浏览器/分辨率 | 2 |
| **合计** | | **79** |

### 1.2 测试用例编号规范

```
TC-{类型}-{模块}-{序号}
类型: FT(功能) / VAL(校验) / BIZ(业务) / UI(界面) / DB(数据库) / API(接口) / WB(白盒) / UT(单元) / PERF(性能) / SEC(安全) / COMP(兼容)
模块: LAB(实验室) / SCH(排期) / RSV(预约) / RVW(审核) / USR(用户) / AUTH(认证)
```

|--------|-----|------|

## 2. 测试用例设计方法

### 2.1 方法应用矩阵

| 方法 | 适用场景 | 应用用例 |
|------|----------|----------|
| 等价类划分 | 输入域划分有效/无效等价类 | VAL-001~012 |
| 边界值分析 | 数值范围边界（min/max/min-1/max+1） | VAL-003, VAL-004, FT-003 |
| 场景法 | 模拟真实业务操作流程 | FT-001~015, BIZ-001~003 |
| 错误推测法 | 基于经验推测可能错误 | VAL-009~012, SEC-001~003 |
| 判定表 | 复杂逻辑判断（审核状态流转） | BIZ-004~008 |
| 因果图 | 多条件组合 | BIZ-007 |
| 正交实验 | 多因素多水平组合 | VAL-012 |
| 语句覆盖 | 每条语句至少执行一次 | WB-001 |
| 判定覆盖 | 每个判定的真/假分支各一次 | WB-002 |
| 条件覆盖 | 每个条件的所有可能取值 | WB-003 |
| 路径覆盖 | 所有独立执行路径 | WB-004 |

### 2.2 测试用例编写规范

每个测试用例包含以下字段：

| 字段 | 说明 |
|------|------|
| **用例编号** | 唯一标识，格式：TC-{类型}-{模块}-{序号} |
| **用例标题** | 简洁描述测试目标 |
| **所属模块** | M1~M5 |
| **优先级** | P0(致命)/P1(高)/P2(中)/P3(低) |
| **前置条件** | 执行测试前必须满足的条件 |
| **测试数据** | 具体的输入数据 |
| **测试步骤** | 详细的操作步骤 |
| **预期结果** | 期望的系统行为 |
| **测试方法** | 等价类/边界值/场景法/错误推测法等 |
| **实际结果** | 执行后填写 |
| **测试结论** | Pass / Fail |
| **缺陷编号** | 如Fail，关联缺陷ID |
