> 从历史报告的 FT / VAL / BIZ / API / SEC 表中提炼的 **20 条高价值手工用例**。
> 每条用例确定可执行,逐条判定。
>
> **关于"历史结论"列**:依据 [`docs/06-reports/09_测试结果汇总.md`](../../docs/06-reports/09_测试结果汇总.md)
> 与 [`docs/05-defects/T29_缺陷清单.md`](../../docs/05-defects/T29_缺陷清单.md) 如实标注上一轮真实执行的通过/失败,
> **仅供对照**。部分用例的依据 ID(如 FT-RVW-004、BIZ-13)取自
> [`docs/03-test-design/测试用例总表.md`](../../docs/03-test-design/测试用例总表.md)(已在行内注明"总表"),与 09 报告中同名条目含义可能不同。
>
> ⚠️ **历史 vs 现状(重要)**:09 报告里标 **FAIL** 的 BUG-001/002/003/005/006/010 相关用例,
> **当前被测源码已修复**(见 [`tests/manual/README.md` §6 缺陷速查](README.md#六已知缺陷速查手测时重点观察))。
> 因此这些用例本轮预期应**无法复现历史失败**,作为**回归守护点**;仍标 FAIL 的是 BUG-007/008/011/012(未修复)。
> 唯 BUG-001 因 insert 路径 SQL(`id != NULL`)陷阱结果存疑,务必实测。**请独立执行并把本轮结果记入**
> [`execution-log-template.md`](execution-log-template.md)。

## 通用前置

- 被测系统已通过 `./start.sh` 启动;前端 http://localhost:81 / 后端 http://localhost:8080。
- 默认账号 `admin` / `admin123`(登录带图形验证码)。
- 接口直测可用 curl / Postman,先登录拿 token 放入 `Authorization` 头;`@Anonymous` 接口无需 token。
- 时间段编码:1=08:00-10:00、2=10:00-12:00、3=14:30-16:30、4=16:30-18:30。
