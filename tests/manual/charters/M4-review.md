# 探索式测试 Charter — M4 预约审核

> 基于会话的探索式测试(SBET)。带着 mission 在时间盒内自由探索。
> 实际结果填入 [`../execution-log-template.md`](../execution-log-template.md)。

## Mission(任务)

探索教师审核流程,验证状态流转(待审核→通过/拒绝)、审核通过的副作用(删除排期、自动拒绝冲突),
并重点考察**重复审核保护**与**冲突自动拒绝**两条业务规则。

## 时间盒

60 分钟(建议)。

## 覆盖区域

- 前端:审核页 `/reservation_review`(待审核列表、审核弹窗:通过/拒绝+审核意见)。
- 接口:`PUT /reservation/reservation/review`(status、id、reviewNote)。
- 数据:`reservation`(status/teacherId/reviewTime)、`lab_schedule`(审核通过后删除)。

## 测试点子(取自真实用例)

- 审核通过:status→1,teacherId 与 reviewTime 被设置(FT-RVW-001、BIZ-011/012)。
- 审核拒绝:status→2,reviewNote 保存(FT-RVW-002)。
- 通过后自动删除对应 lab_schedule 排期记录(FT-RVW-003、BIZ-006)。
- 通过后自动拒绝同 lab 同时段其他待审核预约,但不拒绝自身(FT-RVW-004~006、BIZ-007/016)。
- 重复审核:对已通过(status=1)/已拒绝(status=2)的预约再次审核应被拒绝(FT-RVW-007/008)。
- 审核校验:id=null →"预约ID不能为空";id=99999 →"预约记录不存在"(FT-RVW-009/010、VAL-RVW-001/002)。

## 已知缺陷观察点

> 状态约定:下列"已修复"依据**当前被测源码**(`ReservationController.reviewReservation` 顶部注释:
> "修复 BUG-004 自动拒绝冲突、BUG-010 防止重复审核");历史状态见 [缺陷清单](../../../docs/05-defects/T29_缺陷清单.md)。

- **BUG-004 审核后未自动拒绝冲突预约**(已修复):构造同 lab+同时段的 2 条待审核预约,
  通过其中 1 条,**回归确认**另 1 条应自动变为已拒绝(status=2);若仍停留在待审核则为回退
  (历史曾未修复,对应总表 TC-BIZ-007 / TC-FT-RVW-004,取自测试用例总表)。
- **BUG-010 可重复审核已审预约**(已修复):对 status=1 的预约再次审核,
  应被拒绝而非状态被覆盖(FT-RVW-007 / BIZ-08)。回归确认。
- 审核通过后核对 `lab_schedule` 对应记录确已删除(BIZ-006 / DB-03)。

## 启发式提示

- 自动拒绝冲突需要先在 M3 制造"同 lab + 同 date + 同 slot 的多条待审核",再来 M4 审核。
- 用状态机视角:0→1、0→2 应允许;1→任意、2→任意 应被拒绝(重复审核保护)。
- 通过审核后,横向检查"排期删除 + 冲突自动拒绝"两个副作用是否都发生。
