# 冒烟测试清单 — Smoke Checklist

> 用途:被测系统启动后**立即执行**的快速健康检查。约 12-15 步,5-10 分钟。
> 目标:确认核心链路(登录 → 浏览实验室 → 查看排期 → 提交预约 → 审核 → 退出)整体可用,
> 再决定是否进入完整手测。**任一关键步失败,先排查环境,不要继续完整测试。**

## 前置:启动与连通

- [ ] **S0-1** 仓库根目录执行 `./start.sh`(或 `start.ps1` / `start.bat`),无报错退出。
  - 预期:脚本完成,提示服务已启动。
- [ ] **S0-2** `docker ps` 显示 MySQL、Redis、后端、前端四个容器状态均为 `Up`。
  - 预期:四个容器在线。
- [ ] **S0-3** 浏览器访问 http://localhost:81,登录页正常加载。
  - 预期:出现用户名/密码/验证码输入框与登录按钮(对应 UI-006)。

---

## 核心链路:登录 → 浏览 → 排期 → 预约 → 审核 → 退出

- [ ] **S1 登录** 输入 `admin` / `admin123` + 正确图形验证码,点击登录。
  - 预期:登录成功,跳转首页,左侧菜单加载(对应 FT-USR-001 / API-01)。
- [ ] **S2 验证码校验** 故意输入错误验证码再登录一次。
  - 预期:提示验证码错误,不放行(对应 FT-USR-004)。
- [ ] **S3 浏览实验室** 进入实验室浏览页(`/labs_browse`)。
  - 预期:表格展示实验室,含名称/地点/容量/状态(对应 FT-03 / UI-001)。
- [ ] **S4 实验室详情** 点击任一实验室查看详情。
  - 预期:返回完整实验室信息(对应 FT-RSV 关联 / API-03)。
- [ ] **S5 发布排期** 以管理员发布一条明天的可预约排期(选实验室 + 时间段)。
  - 预期:发布成功,生成 lab_schedule 记录(对应 FT-06 / FT-SCH-001)。
- [ ] **S6 查看今日/排期** 打开排期/今日可预约页(`/apply`)。
  - 预期:按实验室分组显示可用时段(对应 FT-09 / UI-002)。
- [ ] **S7 时间段显示一致性** 核对搜索栏与表格的时间段文案。
  - 预期:文案应一致(如 14:30-16:30);若不一致则记录(关联 BUG-009,状态待复测确认)。
- [ ] **S8 提交预约** 选一个可用时段,填写用途,提交一条**明天**的预约。
  - 预期:提交成功,状态为"待审核"(status=0)(对应 FT-10 / FT-RSV-001)。
- [ ] **S9 查看我的预约** 进入我的预约记录页(`/reservation_records`)。
  - 预期:能看到刚提交的待审核预约(对应 UI-009 / FT-12)。
- [ ] **S10 审核-通过** 进入审核页(`/reservation_review`),对刚才的预约点"通过"。
  - 预期:状态变为"已通过"(status=1),对应排期记录被删除(对应 FT-14 / FT-RVW-001)。
- [ ] **S11 审核-拒绝** 再提交一条预约并对其点"拒绝"+填写意见。
  - 预期:状态变为"已拒绝"(status=2)(对应 FT-15 / FT-RVW-002)。
- [ ] **S12 状态标签** 核对列表中状态标签颜色。
  - 预期:待审核=黄、已通过=绿、已拒绝=红(对应 UI-012)。
- [ ] **S13 退出登录** 右上角头像菜单 → 退出登录。
  - 预期:token 失效,回到登录页(对应 FT-USR-010 / UI-018)。
