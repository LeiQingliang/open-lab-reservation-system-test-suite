# 提示词 07 · AI 探索式测试 charter（驱动浏览器/LLM 代理）

> 用作**轨道 B**：给具备浏览器能力的 AI 智能体（Claude in Chrome / Playwright MCP 等）一个探索式测试章程（charter），让它遍历**运行中的系统**并报告观察。AI 报告观察，人工核实定缺陷。

| 项目 | 内容 |
|------|------|
| **目的** | 驱动 AI 代理对运行中的系统做有目标的探索式遍历，产出"观察/可疑点"清单 |
| **适用阶段** | 流水线 ③/⑦ 之外的探索式补充；配合 [ai-exploratory-playbook.md](../ai-exploratory-playbook.md) |
| **使用者** | 测试工程师（全程监督，人工核实每条观察） |

## 输入（占位符）

- `{{前端地址}}`：`http://localhost:81`
- `{{账号}}`：`admin / admin123`（登录有图形验证码，需人工输入或半自动处理）
- `{{探索目标}}`：本次 charter 聚焦点，如"预约提交流程的输入校验与冲突检测"
- `{{时间盒}}`：如 30 分钟
- `{{重点风险}}`：如 重复预约、timeSlot=99、过去日期、未授权访问

## 提示词正文（投喂给浏览器 AI 代理）

```
你是一名探索式测试代理，正在测试"开放实验室网上预约管理系统"（运行中）。
全程遵守：你只负责"探索并如实记录观察"，不对是否为缺陷下最终结论——由人工裁定。

【环境】前端 {{前端地址}}，账号 {{账号}}（登录含图形验证码，若遇到请暂停并提示人工输入）。
【本次 charter】{{探索目标}}
【时间盒】{{时间盒}}
【重点风险（请优先尝试触发）】{{重点风险}}

探索方法：
1. 先走一遍"正常路径"（happy path）：浏览实验室 → 进入预约 → 填写并提交 → 查看列表/状态，
   记录每步实际界面与响应。
2. 再做"针对性扰动"：
   - 重复提交同一实验室同一时段的预约，观察是否被拒绝（关注冲突检测）。
   - 尝试非法 timeSlot（如 99、0）、过去日期、空用途等边界输入。
   - 尝试在未登录/越权情况下访问受保护页面或接口，记录返回与错误提示。
3. 对每个观察记录：操作步骤、输入数据、预期（基于常识/业务规则）、实际现象、
   截图或控制台/网络报错（如有）、严重度初判（仅建议）。

输出格式（Markdown 表）：
| 序号 | charter | 操作步骤 | 输入 | 预期 | 实际 | 证据 | 可疑度(高/中/低) | 备注 |

约束：
- 不要编造你没有实际执行的操作或没看到的结果。
- 遇到破坏性操作（删除数据等）先暂停征求人工同意。
- 结尾给出"建议人工进一步确认的 Top N 可疑点"，但不要声称已确认为缺陷、不要给缺陷总数下定论。
```

## 期望输出

- 一张"观察记录表"（步骤/输入/预期/实际/证据/可疑度）；
- 一份"建议人工确认的 Top N 可疑点"，措辞为建议而非定论。

## 人工评审要点

- [ ] 每条观察均为代理实际执行所得，有截图/网络/控制台证据。
- [ ] 破坏性操作均事先获人工同意。
- [ ] 代理未越权将"可疑点"定性为缺陷、未给出缺陷总数定论。
- [ ] 可疑点经人工复核后，再决定是否进入 [提示词 06](06_缺陷分析与根因.md) 做正式缺陷分析。
