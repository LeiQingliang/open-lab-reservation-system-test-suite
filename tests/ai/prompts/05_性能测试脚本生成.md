# 提示词 05 · 性能测试脚本生成

> 用 LLM 生成可在本地复跑的性能摸底脚本（curl/shell 基线 + JMeter 计划骨架）。脚本是"可供本地复测的模板"，**不代表已经跑出某性能数字**。

| 项目 | 内容 |
|------|------|
| **目的** | 生成针对真实 API 的性能脚本骨架，便于本地测量响应时间/并发表现 |
| **适用阶段** | 流水线 ④ 用例/代码生成（性能子线） |
| **使用者** | 测试工程师（人工评审 → 本地运行采集真实数据） |

## 输入（占位符）

- `{{基础地址}}`：后端 `http://localhost:8080`
- `{{登录信息}}`：`admin / admin123`（注意 RuoYi 登录带图形验证码，自动化登录需处理 captcha/uuid）
- `{{目标接口}}`：如 `GET /labs_browse/labs_browse/list`（@Anonymous）、`GET /labs_browse/labs_browse/{id}`、`POST /reservation/reservation/insert`
- `{{并发档位}}`：如 1 / 10 / 50 / 100
- `{{脚本形态}}`：`curl+shell` 或 `JMeter .jmx`

## 提示词正文

```
你是一名性能测试工程师。请生成 {{脚本形态}} 的性能测试脚本骨架。

【基础地址】{{基础地址}}
【登录】{{登录信息}}（注意：RuoYi /login 需 username,password,code,uuid，
        其中 code 为图形验证码、uuid 关联 Redis，自动化需先调 /captchaImage 再识别/复用）
【目标接口】{{目标接口}}
【并发档位】{{并发档位}}
【需采集指标】平均/中位/P95/P99 响应时间、成功率、QPS

要求：
1. 脚本参数化（地址、并发数、循环数、token 均为变量）。
2. 优先覆盖无需鉴权的 @Anonymous 接口（如 labs_browse/list）作为基线，便于免登录复跑。
3. 对需鉴权接口，给出"先登录拿 token 再压测"的步骤，并显式标注 captcha 处理为人工/半自动环节。
4. 输出结果到文件，方便归档；脚本内注释用简体中文。
5. 严禁在脚本或注释中写入任何"预期/已得到的性能数值"——数值必须由实际运行产生。
   可在文件头注明"运行后请将真实结果回填到 docs/06-reports/"。

输出：完整脚本 + 一段"运行前置条件与免责说明（脚本未自带任何结果数据）"。
```

## 期望输出

- 参数化的 curl/shell 或 JMeter 脚本骨架；
- 明确的运行前置（登录/验证码处理）与"脚本不含结果数据"的声明。

## 人工评审要点

- [ ] 脚本不含任何杜撰的性能数字；结果须本地实测回填。
- [ ] 真实 API 路径正确；@Anonymous 基线接口可免登录复跑。
- [ ] 验证码/uuid 处理被如实标注（不假装已自动绕过）。
- [ ] 指标定义清晰、结果可归档。
- [ ] 与 `docs/04-test-implementation/08_性能测试脚本.md` 风格一致。
