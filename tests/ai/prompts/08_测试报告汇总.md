# 提示词 08 · 测试报告汇总

> 用 LLM 把**人工确认的真实执行数据**整理成结构化报告草稿。AI 只做编排与排版，**不得杜撰任何数字**。

| 项目 | 内容 |
|------|------|
| **目的** | 将真实测试数据整理为统计表、文字版图表与结论草稿，供人工定稿 |
| **适用阶段** | 流水线 ⑧ 报告汇总 |
| **使用者** | 测试工程师（人工逐项核对数字后定稿） |

## 输入（占位符）

- `{{执行数据}}`：人工执行的真实数据（计划/执行/通过/失败/缺陷数等）
- `{{分类明细}}`：按测试类型/模块的统计明细
- `{{缺陷统计}}`：等级分布、根因分类、缺陷密度
- `{{报告结构}}`：对齐 `docs/06-reports/09_测试结果汇总.md` 的章节结构

## 提示词正文

```
你是一名测试报告撰写助手。请把我提供的"真实执行数据"整理成结构化测试报告草稿。

【真实执行数据】
{{执行数据}}
【分类明细】
{{分类明细}}
【缺陷统计】
{{缺陷统计}}
【报告结构】
{{报告结构}}

要求：
1. 只使用我提供的数字，禁止新增、估算或"补全"任何数值。
   若某项数据缺失，写"待补充"，不要自行编造。
2. 生成：执行概况表、各测试类型统计表、模块统计表、缺陷等级/根因/密度表、
   文字版柱状图（用字符块表示比例）、测试结论。
3. 结论要基于数据，不夸大；明确指出未达成的准出条件。
4. 用简体中文，Markdown 表格；风格对齐 docs/06-reports 的现有报告。
5. 文末注明"本报告数据来自人工执行，AI 仅协助整理排版"。

诚信约束：你不能声称 AI 自主执行了测试或自主发现了缺陷；
所有结果均为人工执行所得，你只是整理者。
```

## 期望输出

- 结构化报告草稿（统计表 + 文字版图表 + 结论）；
- 缺失项标注"待补充"，文末注明 AI 仅协助整理。

## 人工评审要点

- [ ] 报告中**每个数字**都能在真实执行数据中找到来源，无 AI 杜撰。
- [ ] 缺失数据标为"待补充"，未被自动补全。
- [ ] 结论与数据一致，未夸大；如实列出未达成的准出条件。
- [ ] 与 `docs/06-reports/09_测试结果汇总.md` 数据完全一致。
- [ ] 文末"AI 仅协助整理"声明保留。
