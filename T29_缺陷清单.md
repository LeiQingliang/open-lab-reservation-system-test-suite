## 严重缺陷

### BUG-001: 预约提交无冲突检测
| 字段 | 内容 |
|------|------|
| **缺陷编号** | T29_BUG-RSV-001 |
| **缺陷标题** | insertReservation接口未进行预约冲突检测，允许同一用户在同一实验室同时段重复提交预约 |
| **所属模块** | M3 预约管理 |
| **严重程度** | 严重 |
| **优先级** | 高 |
| **状态** | 已修复 |
| **发现版本** | labs-3.8.9 |
| **测试环境** | Windows 11 + JDK21 + MySQL8.0 |
| **前置条件** | 已登录，实验室1在2026-06-22时段1有排期 |
| **复现步骤** | 1. POST /reservation/reservation/insert {userId:1,labId:1,date:"2026-06-22",timeSlot:1} 2. 再次提交相同的请求 |
| **实际结果** | 两次请求均返回"操作成功"(code:200)，数据库中存在两条重复预约记录(ID=1和ID=2) |
| **预期结果** | 第二次提交应被拒绝，提示"该时段已被预约" |
| **原因分析** | ReservationController.insertReservation()方法中仅设置了status=0L，未调用selectConflictingReservations()进行冲突检测 |
| **修复方案** | 在insertReservation方法中增加冲突检测逻辑 |
| **修改文件** | ReservationController.java |
| **回归结果** | 通过 |

### BUG-002: 时间段范围未校验
| 字段 | 内容 |
|------|------|
| **缺陷编号** | T29_BUG-RSV-002 |
| **缺陷标题** | 预约提交时未校验timeSlot的取值范围(1-4)，可提交timeSlot=99等非法值 |
| **所属模块** | M3 预约管理 |
| **严重程度** | 严重 |
| **优先级** | 高 |
| **状态** | 已修复 |
| **复现步骤** | POST /reservation/reservation/insert {timeSlot:99} |
| **实际结果** | 预约成功插入数据库(time_slot=99) |
| **预期结果** | 返回错误提示"时间段必须在1-4之间" |
| **原因分析** | Controller层和Service层均未对timeSlot进行范围校验 |
| **修复方案** | 增加timeSlot范围校验(1-4) |
| **回归结果** | 通过 |

### BUG-003: 过去日期未拦截
| 字段 | 内容 |
|------|------|
| **缺陷编号** | T29_BUG-RSV-003 |
| **缺陷标题** | 允许提交2020-01-01等过去日期的预约 |
| **所属模块** | M3 预约管理 |
| **严重程度** | 严重 |
| **优先级** | 高 |
| **状态** | 未修复 |
| **复现步骤** | POST /reservation/reservation/insert {reservationDate:"2020-01-01"} |
| **实际结果** | 预约成功 |
| **预期结果** | 提示"不能预约过去日期" |
| **原因分析** | 未进行日期与当前日期的比较 |
| **修复方案** | 增加预约日期>=当前日期的校验 |
| **备注** | 留待后续修复 |

### BUG-004: 审核后未拒绝其他冲突预约
| 字段 | 内容 |
|------|------|
| **缺陷编号** | T29_BUG-RSV-004 |
| **缺陷标题** | 审核通过某预约后，未自动拒绝同一实验室同时段的其他待审核预约 |
| **所属模块** | M4 预约审核 |
| **严重程度** | 严重 |
| **优先级** | 中 |
| **状态** | 未修复 |
| **原因分析** | reviewReservation方法仅删除了lab_schedule排期记录，未更新其他冲突的reservation记录状态 |
| **修复方案** | 审核通过时，批量将冲突的待审核预约状态设为"已拒绝" |

## 一般缺陷

### BUG-005: 不存在的实验室可预约
提交labId=99999（不存在的实验室ID）的预约成功

### BUG-006: 空用途可提交
purpose=""可成功提交

### BUG-007: 排期发布允许过去日期
发布排期接口接受2020-01-01等过去日期

### BUG-008: 实验室删除无关联检查
删除实验室后，关联的预约记录和排期记录未做级联处理

### BUG-009: 前端时间显示不一致
apply页面搜索栏显示"14:00-16:00"，表格显示"14:30-16:30"

### BUG-010: 审核心得重复审核
已审核通过的预约可再次审核通过，状态被覆盖

## 轻微缺陷

### BUG-011: 排期重复发布无去重
同一实验室同一日期同一时段可多次发布

### BUG-012: 操作日志中密码字段可能泄露
部分日志记录中包含了请求参数的完整JSON，可能包含敏感信息
