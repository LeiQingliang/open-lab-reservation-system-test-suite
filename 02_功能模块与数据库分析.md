### 3.4 核心业务逻辑 — publishAvailableSlots

```java
// LabScheduleServiceImpl.java — 源码
@Override
@Transactional  // ✅ 有事务保护
public void publishAvailableSlots(Date date, List<Integer> timeSlots, List<Long> labIds) {
    if (date == null || timeSlots == null || labIds == null) {
        throw new ServiceException("参数不能为空");
    }
    // 笛卡尔积插入：labIds × timeSlots
    for (Long labId : labIds) {
        for (Integer timeSlot : timeSlots) {
            LabSchedule schedule = new LabSchedule();
            schedule.setLabId(labId);
            schedule.setDate(date);
            schedule.setTimeSlot(timeSlot.longValue());
            schedule.setIsAvailable(1);
            labScheduleMapper.insertLabSchedule(schedule);
        }
    }
}
```

**DTO校验 (PublishScheduleRequest)**:
```java
@NotNull(message = "日期不能为空")
@FutureOrPresent(message = "日期必须为今天或将来")  // ✅ 已校验过去日期
private LocalDate date;

@NotEmpty(message = "至少需要选择一个时段")
private List<Integer> timeSlots;

@NotEmpty(message = "至少需要选择一个实验室")
```

**已实现的业务规则**:

## 5. M4 预约审核模块

### 5.1 功能描述
教师审核用户的预约申请，支持"通过"(status=1)和"拒绝"(status=2)。审核通过时自动删除冲突的lab_schedule排期记录。

### 5.2 核心业务逻辑 — reviewReservation

```java
// ReservationController.java — 源码
@PutMapping("/review")
public AjaxResult reviewReservation(@RequestBody Reservation reservation) {
    Long teacherId = getUserId();
    reservation.setTeacherId(teacherId);

    if (reservation.getStatus() != null && reservation.getStatus() == 1) {
        // ✅ 审核通过时删除lab_schedule中冲突时间段的记录
        labScheduleService.deleteByReservation(reservation);
    }
    return toAjax(reservationService.reviewReservation(reservation));
}
```

**审核流程**:
```
用户提交预约(status=0)
    ↓
教师审核
    ├── 通过(status=1) → 删除lab_schedule冲突排期 + 更新reservation
    └── 拒绝(status=2) → 仅更新reservation状态
```

**已发现的缺陷**:
- ⚠️ 审核通过后未自动拒绝同一实验室同时段的其他待审核预约 (BUG-004)
- ⚠️ 已审核通过的预约可再次审核 (BUG-010)

### 5.3 ReservationServiceImpl.reviewReservation 源码分析

```java
@Override
public int reviewReservation(Reservation reservation) {
    reservation.setReviewTime(Timestamp.valueOf(LocalDateTime.now())); // ✅ 设置审核时间
    reservationMapper.updateReviewStatus(reservation);  // 更新status+teacherId+reviewTime
    return reservationMapper.reviewReservation(reservation); // 更新status+teacherId+reviewTime+reviewNote
}
```

⚠️ **问题**: `updateReviewStatus` 和 `reviewReservation` 两次更新同一条记录，存在冗余；且未做审核前置状态检查。

## 9. 核心业务表详细分析

### 9.1 labs 表

```sql
CREATE TABLE labs (
    id          BIGINT       PRIMARY KEY AUTO_INCREMENT COMMENT '实验室ID',
    lab_name    VARCHAR(255) COMMENT '实验室名称，如"软件工程实验室"',
    location    VARCHAR(255) COMMENT '实验室地址，如"教学楼A305"',
    capacity    BIGINT       COMMENT '实验室可容纳人数',
    description TEXT         COMMENT '实验室简介或备注',
    status      INT          DEFAULT 0 COMMENT '状态(0正常 1停用)',
    created_at  DATETIME     COMMENT '创建时间',
    updated_at  DATETIME     COMMENT '最后更新时间'
) ENGINE=InnoDB;
```

## 11. Mapper XML SQL分析

### 11.1 ReservationMapper.xml — 最复杂的SQL

**多表联查 (selectReservationList)**:
```xml
SELECT r.*,
       u.nick_name AS nickName,      -- 申请人昵称
       l.lab_name AS labName,         -- 实验室名称
       t.nick_name AS teacherNickName -- 教师昵称
FROM reservation r
LEFT JOIN sys_user u ON r.user_id = u.user_id
LEFT JOIN labs l ON r.lab_id = l.id
LEFT JOIN sys_user t ON r.teacher_id = t.user_id
```

**动态查询条件**:
- `userId`, `labId`, `status`, `teacherId` — 精确匹配
- `beginReservationDate`, `endReservationDate` — 日期范围

**冲突检测SQL (selectConflictingReservations)**:
```xml
SELECT * FROM reservation
WHERE lab_id = #{labId}
  AND reservation_date = #{reservationDate}
  AND time_slot = #{timeSlot}
  AND status = 0        -- 仅检测待审核状态
  AND id != #{id}       -- 排除自身
```

⚠️ **问题**: 此方法已定义但未在Controller中调用！

## 14. 数据一致性与约束分析

### 14.1 缺失的物理外键

| 子表 | 字段 | 父表 | 风险 |
|------|------|------|------|
| reservation | user_id | sys_user.user_id | 删除用户后预约记录悬空 |
| reservation | lab_id | labs.id | 删除实验室后预约悬空 (BUG-008) |
| reservation | teacher_id | sys_user.user_id | 删除教师后审核信息悬空 |
| lab_schedule | lab_id | labs.id | 删除实验室后排期悬空 |

### 14.2 缺失的唯一约束

| 表 | 字段组合 | 风险 |
|----|----------|------|
| lab_schedule | (lab_id, date, time_slot) | 排期重复发布 (BUG-011) |
| reservation | (user_id, lab_id, reservation_date, time_slot, status=0) | 重复预约 (BUG-001) |

### 14.3 事务边界

- `LabScheduleServiceImpl.publishAvailableSlots()` — ✅ `@Transactional`
- `ReservationController.insertReservation()` — ❌ 无事务注解
- `ReservationController.reviewReservation()` — ❌ 无事务注解(Controller层调用两个Service方法，无原子性保证)
