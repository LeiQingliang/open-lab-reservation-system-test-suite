## 三、后端模块架构

Maven 多模块项目，`pom.xml` 声明 5 个子模块：

```
labs (父POM, pom packaging)
├── labs-admin       — Web入口 + 所有Controller (21个)
├── labs-framework   — 核心框架 (安全/配置/拦截器/工具类)
├── labs-system      — 系统管理模块 (RBAC: 用户/角色/菜单/部门...)
├── labs-common      — 通用模块 (注解/枚举/异常/工具类)
└── labs-management  — 业务模块 (实验室预约核心业务)
```

### 模块职责详解

#### `labs-admin` — Web入口
- 启动类 `LabsApplication.java`
- 21 个 REST Controller，分布在 4 个路径：

| 路径 | Controller | 数量 |
|------|-----------|------|
| `system/` | 用户/角色/菜单/部门/岗位/字典/配置/通知/登录/注册/个人中心 | 13 |
| `monitor/` | 在线用户/操作日志/登录日志/服务监控/缓存监控 | 5 |
| `common/` | 验证码/通用(文件上传等) | 2 |
| `tool/` | 测试 | 1 |

#### `labs-framework` — 核心框架
安全配置、JWT 过滤器、全局异常处理、数据权限 AOP、Redis 配置等框架级能力。

#### `labs-system` — 系统模块 (60 个 Java 文件)
完整的 RBAC 权限模型：

| 实体 | 说明 |
|------|------|
| `SysUser` | 用户 (含密码/部门/状态/最后登录IP) |
| `SysRole` | 角色 |
| `SysMenu` | 菜单 (含权限标识符如 `system:user:list`) |
| `SysDept` | 部门 (树形结构) |
| `SysPost` | 岗位 |
| `SysUserRole` / `SysRoleMenu` / `SysRoleDept` / `SysUserPost` | 多对多关联 |
| `SysConfig` | 系统参数配置 |
| `SysDictType` / `SysDictData` | 字典管理 |
| `SysNotice` | 通知公告 |
| `SysOperLog` / `SysLogininfor` | 操作日志 / 登录日志 |

#### `labs-management` — 业务模块 (16 个 Java 文件)
三个业务子包：

| 子包 | Controller | 功能 |
|------|-----------|------|
| `labs_browse` | `LabsController` | 实验室信息 CRUD（名称/地址/容纳人数/简介） |
| `apply` | `LabScheduleController` | 排期管理（管理员发布可用时段：日期+时间段+实验室） |
| `reservation` | `ReservationController` | 预约记录管理（含审核：通过/拒绝） |

**预约实体 (`Reservation`) 核心字段：**
- `userId` — 申请人
- `labId` — 实验室
- `reservationDate` — 预约日期
- `timeSlot` — 时间段 (1=8:00-10:00, 2=10:00-12:00, 3=14:30-16:30, 4=16:30-18:30)
- `purpose` — 预约用途
- `status` — 状态 (0=待审核, 1=已通过, 2=已拒绝)
- `teacherId` — 审核教师
- `reviewTime` / `reviewNote` — 审核信息

```

│  管理员                                                    │
```

## 七、安全模型

| 层面 | 实现 |
|------|------|
| **认证** | JWT token (登录后签发，前端存 cookie，每次请求 header 携带) |
| **授权** | Spring Security + `@PreAuthorize` + 权限字符串 (如 `system:user:list`) |
| **数据权限** | `@DataScope` 注解，按部门层级过滤数据 |
| **接口防护** | `@RateLimiter` (限流), `@RepeatSubmit` (防重复提交) |
| **敏感数据** | `@Sensitive` 脱敏注解；密码 BCrypt 加密 |
| **登录安全** | 验证码 (可配置开关)、IP 黑名单、登录日志审计 |

## 八、项目特色与问题

### 特色
1. **成熟框架底子**：基于 RuoYi v3.8.9，自带完整的 RBAC、日志审计、代码生成器、服务监控等企业级能力
2. **代码生成器**：可一键生成 CRUD 前后端代码（Velocity 模板引擎）
3. **Vue3 + Element Plus + Vite**：前端技术栈较新，构建速度快
4. **业务清晰**：实验室预约从发布排期 → 浏览 → 申请 → 审核形成完整闭环

### 潜在问题
1. **Spring Boot 2.5.15 / JDK 1.8** — 版本偏旧，Spring Boot 2.5 已于 2022 年停止维护
2. **JWT 库 `jjwt 0.9.1`** — 2018 年版本，存在已知安全漏洞
3. **前端时间显示不一致** — apply 页面的搜索下拉和表格显示的时间段文字不统一
4. **SQL 缺少业务表** — `labs`/`lab_schedule`/`reservation` 建表语句未在 SQL 文件中，依赖 ORM 自动建表
5. **无测试** — 仅有 `labs-management/src/test/` 目录骨架，未见实际单元测试
6. **前端 `package.json` author 为 "santh"**，与 `pom.xml` description 一致，说明是 RuoYi 基础上的二次开发

## 补充分析：业务表DDL（从Mapper XML逆向恢复）

由于 `ry_20250417.sql` 仅包含若依框架19张系统表，3张核心业务表（`labs`, `lab_schedule`, `reservation`）的DDL未在SQL脚本中明确定义。以下从 MyBatis Mapper XML 和 Java 实体类逆向恢复：

### labs 表（实验室信息）

```sql
CREATE TABLE labs (
    id          BIGINT(20)   NOT NULL AUTO_INCREMENT  COMMENT '实验室ID',
    lab_name    VARCHAR(100) NOT NULL                 COMMENT '实验室名称，如"软件工程实验室"',
    location    VARCHAR(200) DEFAULT NULL             COMMENT '实验室地址，如"教学楼A305"',
    capacity    BIGINT(20)   DEFAULT NULL             COMMENT '实验室可容纳人数',
    status      INT          DEFAULT 0                COMMENT '状态（0正常 1停用）',
    description VARCHAR(500) DEFAULT NULL             COMMENT '实验室简介或备注',
    created_at  DATETIME     DEFAULT NULL             COMMENT '创建时间',
    updated_at  DATETIME     DEFAULT NULL             COMMENT '最后更新时间',
    PRIMARY KEY (id)
) ENGINE=InnoDB COMMENT='实验室信息表';
```

### lab_schedule 表（排期信息）

```sql
CREATE TABLE lab_schedule (
    id            BIGINT(20) NOT NULL AUTO_INCREMENT  COMMENT '排班记录ID',
    lab_id        BIGINT(20) NOT NULL                 COMMENT '实验室ID（关联labs表）',
    date          DATE       NOT NULL                 COMMENT '可预约日期',
    time_slot     BIGINT(20) NOT NULL                 COMMENT '时间段编号（1:8-10,2:10-12,3:14:30-16:30,4:16:30-18:30）',
    is_available  INT        DEFAULT 1                COMMENT '是否可用（1可用 0不可用）',
    note          VARCHAR(500) DEFAULT NULL           COMMENT '备注',
    PRIMARY KEY (id),
    INDEX idx_lab_date (lab_id, date)
) ENGINE=InnoDB COMMENT='实验室排期表';
```

### reservation 表（预约记录）

```sql
CREATE TABLE reservation (
    id                BIGINT(20)   NOT NULL AUTO_INCREMENT  COMMENT '预约记录ID',
    user_id           BIGINT(20)   NOT NULL                 COMMENT '申请人ID（关联sys_user表）',
    lab_id            BIGINT(20)   NOT NULL                 COMMENT '实验室ID（关联labs表）',
    reservation_date  DATE         NOT NULL                 COMMENT '预约日期',
    time_slot         BIGINT(20)   NOT NULL                 COMMENT '预约时间段编号',
    purpose           VARCHAR(500) DEFAULT NULL             COMMENT '预约用途说明',
    status            BIGINT(20)   DEFAULT 0                COMMENT '预约状态（0:待审核,1:已通过,2:已拒绝）',
    teacher_id        BIGINT(20)   DEFAULT NULL             COMMENT '审核教师ID（关联sys_user表）',
    review_time       DATETIME     DEFAULT NULL             COMMENT '审核时间',
    review_note       VARCHAR(500) DEFAULT NULL             COMMENT '审核意见/备注',
    created_at        DATETIME     DEFAULT NULL             COMMENT '提交时间',
    PRIMARY KEY (id),
    INDEX idx_user (user_id),
    INDEX idx_lab_date (lab_id, reservation_date),
    INDEX idx_status (status)
) ENGINE=InnoDB COMMENT='实验室使用记录表';
```
