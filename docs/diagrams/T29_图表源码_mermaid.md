## 图2-1 核心业务流程图
```mermaid
flowchart TB
    A[学生登录系统] --> B[浏览实验室/查看可预约时段] --> C[提交预约：时段/日期/用途校验+冲突检测]
    C --> D{校验与冲突是否通过?}
    D -- 否 --> E[驳回并提示错误]
    D -- 是 --> F[生成待审核预约 status=0] --> G[教师审核] --> H{审核通过?}
    H -- 否 --> I[预约被拒绝 status=2]
    H -- 是 --> J[预约成功 status=1：释放排期+自动拒绝同时段冲突]
    J --> K[结束]; I --> K
```

## 图4-1 insertReservation() 控制流图
```mermaid
flowchart TB
    S[开始：接收预约请求] --> D1{timeSlot∈[1,4]?}
    D1 -- 否 --> E1[时间段无效]
    D1 -- 是 --> D2{预约日期≥今天?}
    D2 -- 否 --> E2[不能预约过去日期]
    D2 -- 是 --> D3{labId 非空?}
    D3 -- 否 --> E3[实验室ID不能为空]
    D3 -- 是 --> D4{purpose 非空?}
    D4 -- 否 --> E4[预约用途不能为空]
    D4 -- 是 --> D5{存在时段冲突?}
    D5 -- 是 --> E5[该时段已被预约]
    D5 -- 否 --> OK[设置status=0,插入预约] --> R[返回成功]
```

## 图4-2 reviewReservation() 处理流程图
```mermaid
flowchart TB
    S[开始：接收审核请求] --> D1{预约ID为空?}
    D1 -- 是 --> E1[预约ID不能为空]
    D1 -- 否 --> D2{预约记录存在?}
    D2 -- 否 --> E2[预约记录不存在]
    D2 -- 是 --> D3{已审核 status≠0?}
    D3 -- 是 --> E3[不能重复审核]
    D3 -- 否 --> T[取教师ID,设置审核人] --> D4{审核为通过?}
    D4 -- 否 --> J2[拒绝 status=2] --> U[更新审核状态并返回]
    D4 -- 是 --> J1[自动拒绝同时段冲突+删除排期] --> U
```
